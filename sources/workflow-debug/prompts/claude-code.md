Use `workflow-debug` to diagnose a Chromatic git, baseline, pull request, merge queue, or patch-build issue.

Goal:
Identify the primary workflow diagnosis and return one exact next step.

Known evidence:
- Exact Chromatic command or action inputs: <paste>
- CI event type and checkout step: <paste>
- Warning or error text: <paste>
- Expected comparison target: <paste>
- Any relevant git command output: <paste if known>

Instructions:
- Pick one diagnosis branch first.
- Explain the current run truthfully.
- Ask for exactly one next artifact if the evidence is still incomplete.
